# Kagi-Kashi-EX

アプリ制作プロジェクトの拡張版（kagi-kashi-ex）

スマホ画面をつくるやつ

## すること

- 縦並びに表示。
- 開錠。施錠。一旦施錠している場合、メッセージを残せる。
- 部室を持っているサークルを上から表示させる。
- お気に入り機能。
- 通知
- 名前検索。
- 活動報告書。
- 引継ぎ設定。

---

## 主な機能（kagi-kashi-ex）

スマホ画面での鍵の貸し出し・返却・サークル状況管理を行うWebアプリケーションです。

- **サークル一覧表示**: 鍵の保管状況（保管中、活動中、一時施錠中）を縦並びに表示。
- **お気に入り機能**: お気に入り登録したサークルを自動的に最上部にピン留め表示。
- **鍵の借用・返却**: 各サークルに登録されているメンバーのみが借用・返却可能。なりすまし防止・不正操作防止機能を搭載。
- **活動報告書**: メンバーによる活動報告書の作成・提出および履歴表示。
- **サークル管理・引継ぎ設定**: メンバーの追加・削除が可能。
- **セキュリティ（30日退部ロック）**: 盗難防止および監査ログ維持のため、サークル登録から30日間はメンバー登録解除（退部）をブロックする安全設計。
- **ユーザー設定画面 (`/settings`)**: 所属サークルの設定や、お気に入りサークルの一覧確認。

## 画面遷移・操作手順

1. **ログイン**: 学籍番号と氏名を入力してログインします。
2. **一覧画面 (ホーム)**: サークルの一覧（保管中：灰色、一時施錠：赤、活動中：緑）が表示されます。お気に入り登録されているサークルは最上部に表示されます。
3. **詳細画面**: サークルをクリックすると詳細ページへ遷移します。
4. **アクション**: 詳細画面から「鍵を借りる」「鍵を返す」「状況の更新」「活動報告書の提出」「メンバーの管理・引継ぎ設定」が行えます。

## 鍵を借りる場合

- ログイン中のユーザーが、該当するサークルにメンバーとして登録されている必要があります。
- 学籍番号、氏名、鍵番号を入力して借用します（ログイン中の学籍番号と一致している必要があります）。
- 借用すると、サークルのステータスが自動的に「活動中」になります。

## 鍵を返す場合

- ログイン中のユーザーが、該当するサークルのメンバーである必要があります。
- 返却を完了すると、サークルのステータスが「保管中」に戻ります。

## 環境構築方法

### Windows の場合

1. ターミナルを起動し、以下のコマンドを実行します。

```bash
py -3 -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

### Linux / macOS の場合

1. ターミナルを起動し、以下のコマンドを実行します。

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## データベースの初期化について

- **自動初期化**: アプリの起動時にデータベースファイル (`instance/flaskr.sqlite`) または `clubs` テーブルが存在しない場合、自動的にスキーマが実行され、初期シードデータが投入されます。
- **手動初期化**: データをクリアして最初からやり直したい場合は、以下のコマンドを実行します。

```bash
flask init-db
# または仮想環境のPython経由で実行:
.venv/bin/python -m flask init-db
```

## 起動方法

- ディレクトリ直下にある `app.py` を起動します。

```bash
python app.py
```

## ディレクトリ構成

```text
kagi-kashi-ex/
├── .venv/               # 仮想環境フォルダ
├── flaskr/              # アプリケーションのメインパッケージ
│   ├── templates/       # HTMLテンプレートファイル群
│   ├── __init__.py      # Flaskアプリ生成、ルーティング、各種ロジック
│   ├── db.py            # データベース接続管理、自動初期化ロジック、シードデータ
│   └── schema.sql       # データベースのテーブルスキーマ定義
├── instance/            # インスタンスフォルダ（SQLiteデータベースファイルが配置されます）
│   └── flaskr.sqlite
├── tests/               # ユニットテストコード
│   └── test_routes.py   # 各画面・APIの動作およびセキュリティ検証テスト
├── app.py               # アプリケーションのエントリーポイント
├── README.md            # このファイル
└── requirements.txt     # 依存ライブラリ一覧
```
